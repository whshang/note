# Prompt Engineering 实战：从"写长文"到"写好提示词"

刚开始接触 AI 时，我有个误解：Prompt 越长越好，描述越详细越好。

结果呢？AI 经常忽略关键信息，生成的内容偏离主题，上下文消耗巨大。

经过几个月的摸索，我才明白：好 Prompt 不是"写长文"，而是"写好提示词"。

## 第一个误区：写长文

我曾写过一个 500 字的 Prompt：请帮我写一个登录页面。这个页面需要有用户名输入框和密码输入框，还有登录按钮。样式要美观，符合现代设计风格。输入框要有验证功能，比如不能为空，密码要符合复杂度要求。还要有记住密码的选项。页面要响应式，能在手机和电脑上正常显示。按钮要有点击效果，比如按下时变色。错误信息要在输入框下方显示红色文字。整个页面的背景色是白色，输入框边框是灰色。字体大小是 16px。等等...

AI 生成的页面有很多我没提到的功能，缺少我特别强调的验证逻辑，样式和我描述的也不一致。

问题在哪？

信息密度低。500 字里，真正重要的信息只有 50 字，AI 不知道该关注什么。

优先级混乱。验证逻辑是我最关心的，但在文中不起眼，样式细节占了大量篇幅。

上下文浪费。500 字的 Prompt，留给 AI 回答的空间很少。

## 第二个误区：过度精确

吸取教训后，我开始写"精确"的 Prompt：页面结构：div.container > form.login-form > div.input-group > input.username-input[type=text] + input.password-input[type=password] + button.login-btn

AI 严格按照这个结构生成，但代码可读性差，没有考虑实际使用场景，缺少必要的错误处理。

过度精确限制了 AI 的创造性。有时候，"模糊"比"精确"更有效。

## 现在的做法：结构化 Prompt

经过多次试错，我总结了一个结构化 Prompt 框架。

### 框架模板

```
【任务】: [一句话说明要做什么]
【上下文】: [关键背景信息]
【约束】: [必须遵守的规则]
【示例】: [1-2 个具体例子]
【输出】: [期望的输出格式]
```

### 实际应用

用上面的例子，我现在的 Prompt 是：

```
【任务】: 生成一个登录页面的 HTML 和 CSS 代码
【上下文】: 这是一个企业内部系统，用户都是员工，需要高安全性
【约束】:
- 必须包含用户名和密码输入验证
- 必须有记住密码功能
- 必须响应式设计
【示例】:
- 验证：用户名不能为空，密码至少8位
- 样式：简洁商务风格，白色背景，蓝色主题色
【输出】: 返回完整的 HTML 文件，包含样式和简单 JS 验证
```

效果：生成的页面完全符合要求，验证逻辑正确，样式简洁美观，只用了 150 字的 Prompt。

## 几个技巧

### 优先级排序

在 Prompt 中，顺序很重要。

错误顺序：样式要求 → 验证逻辑 → 安全要求

正确顺序：安全要求 → 验证逻辑 → 样式要求

AI 会更关注开头的内容。

### 用示例代替描述

比起文字描述，示例更有效。

文字描述："验证逻辑：用户名不能为空，密码至少8位，不能包含特殊字符"

示例：
```
验证示例：
- 用户名: '' → 错误: '用户名不能为空'
- 密码: '123' → 错误: '密码至少8位'
- 密码: 'password!' → 错误: '不能包含特殊字符'
```

### 明确输出格式

告诉 AI 你想要的输出格式。返回完整的 HTML 文件，CSS 写在 <style> 标签内，JS 写在 <script> 标签内，不要额外解释，只返回代码。

### 分步处理复杂任务

对于复杂任务，分步进行。

第一步：分析需求，列出功能点。

第二步：设计页面结构，返回 HTML 骨架。

第三步：添加样式，返回完整页面。

## 实战案例

### 生成 API 接口文档

之前（500 字长文）：请帮我写一个用户管理系统的 API 接口文档。这个系统有用户注册、登录、信息修改等功能。注册接口需要用户名、密码、邮箱，还要验证邮箱格式。登录接口需要用户名和密码，成功后返回 token。信息修改接口需要 token 验证，可以修改用户名、邮箱等信息...

现在（结构化）：

```
【任务】: 生成用户管理系统的 API 接口文档
【上下文】: 企业内部系统，RESTful API，JSON 格式
【约束】:
- 所有接口需要 token 验证（except login/register）
- 返回格式统一：{success: boolean, data: object, error: string}
【示例】:
POST /api/login
Request: {username: string, password: string}
Response: {success: true, data: {token: string}} | {success: false, error: string}
【输出】: 返回完整的 API 文档，包含所有 CRUD 接口
```

文档更准确，格式更统一，生成时间更短。

## 检查清单

每次写 Prompt 前，我会问自己：

任务明确吗？（一句话能说清楚）

优先级对吗？（最重要的放前面）

约束清晰吗？（必须遵守的规则）

有示例吗？（1-2 个具体例子）

输出格式明确吗？（期望的具体格式）

## 最后想说

好的 Prompt Engineering 不是"写作文"，而是"给指令"。结构化比长篇大论重要，优先级比信息量重要，示例比抽象描述重要，格式化比自由发挥重要。

这套方法帮我提高了 60% 的 AI 使用效率。
