# 技能拆解：把复杂任务拆解成 AI 能完成的步骤

## 背景

刚开始使用 AI 时，我总是遇到这种情况：
"我想让 AI 做一件事，但它说不理解。"

后来我发现，问题在于：
**我把复杂任务当成单一任务给 AI，但 AI 只能处理相对简单的步骤。**

## 我的第一个误区：给 AI 复杂任务

### 实际例子

我想让 AI 帮我"分析用户反馈，找出产品问题，提出改进建议"。

我给 AI 的提示是：
> "这里有 1000 条用户反馈，请分析找出产品问题并提出改进建议。"

### 结果

AI 给出的回答：
- "大部分用户反馈产品很好"
- "建议继续努力"
- "没有发现明显问题"

但实际上，这 1000 条反馈里有很多具体的问题。

### 问题在哪

1. **任务太复杂**
   - 同时要求分析 + 总结 + 建议
   - AI 不知道从哪里开始

2. **缺少明确步骤**
   - 没有告诉 AI 具体怎么分析
   - 没有定义什么叫"问题"

3. **期望不现实**
   - 1000 条反馈需要人工分析，AI 也需要分步处理

## 我的第二个误区：拆得太细

吸取教训后，我开始拆解任务，但拆得太细：

1. 统计反馈数量
2. 分类反馈类型  
3. 统计每类数量
4. 找出最多的类别
5. 分析该类别的具体内容
6. 提取关键词
7. 归纳问题
8. 提出建议

### 结果

AI 按步骤执行，但：
- 效率很低，要执行 8 次
- 每步之间的衔接不好
- 最终结果不如预期

### 问题在哪

拆解过度让 AI 失去了"整体思考"的能力。

## 我现在的做法：合理拆解 + 分步执行

经过多次实践，我总结了一个技能拆解框架：

### 框架模板

```
【主任务】: [整体目标]
【子任务分解】: [3-5 个关键步骤]
【每步要求】: [该步骤的具体要求]
【衔接逻辑】: [步骤间的依赖关系]
【最终整合】: [如何合并结果]
```

### 实际应用

用上面的用户反馈例子，我现在的做法：

```
【主任务】: 分析 1000 条用户反馈，找出产品问题并提出改进建议
【子任务分解】:
1. 反馈分类（技术问题/功能问题/体验问题/其他）
2. 问题统计（统计各类问题的数量和关键词）
3. 深度分析（对高频问题进行根因分析）
4. 改进建议（基于分析结果提出具体建议）
【每步要求】:
- 步骤 1: 每条反馈分类，输出 [反馈ID, 分类]
- 步骤 2: 统计各分类数量，提取 Top10 关键词
- 步骤 3: 对 Top3 问题分析原因和影响
- 步骤 4: 提供可执行的改进建议
【衔接逻辑】:
- 步骤 1 → 步骤 2: 基于分类统计
- 步骤 2 → 步骤 3: 基于高频问题分析
- 步骤 3 → 步骤 4: 基于根因提建议
【最终整合】: 生成问题报告 + 改进计划
```

### 效果

现在 AI 能：
- 有序地完成每个步骤
- 步骤间逻辑清晰
- 最终结果完整可用

## 关键技巧

### 1. 拆解原则

**3-5 个步骤原则**：
- 太少：任务还是太复杂
- 太多：步骤间衔接困难
- 3-5 个：平衡复杂度和可操作性

### 2. 步骤独立性

每个步骤应该是相对独立的：

```
✅ 好的拆解：
步骤 1: 数据清洗
步骤 2: 数据分析  
步骤 3: 结果可视化

❌ 坏的拆解：
步骤 1: 清洗姓名列
步骤 2: 清洗年龄列
步骤 3: 清洗性别列
步骤 4: 分析姓名分布
步骤 5: 分析年龄分布
```

### 3. 明确输出格式

每步都要明确输出格式：

```
步骤 1 输出: CSV 格式，[ID, 清洗后数据, 状态]
步骤 2 输出: JSON 格式，{"统计结果": {...}}
步骤 3 输出: 图表 + 分析报告
```

### 4. 设置检查点

在关键步骤后设置检查：

```
步骤 1: 数据清洗
   ↓
检查点 1: 验证清洗结果（数据量、格式、异常值）
   ↓
步骤 2: 数据分析
   ↓
检查点 2: 验证分析结果（合理性、完整性）
   ↓
步骤 3: 结果输出
```

## 实战案例

### 案例：让 AI 写技术文档

**之前**（单一任务）：
> "请帮我写一份关于这个 API 的技术文档。"

**现在**（拆解步骤）：
```
【主任务】: 编写 API 技术文档
【子任务分解】:
1. 接口分析（提取所有端点和参数）
2. 功能说明（描述每个接口的作用）
3. 使用示例（提供调用示例和返回结果）
4. 错误处理（列出可能的错误和解决方案）
5. 文档整合（生成完整文档）
【每步要求】:
- 步骤 1: 输出接口清单表格
- 步骤 2: 为每个接口写 1-2 句说明
- 步骤 3: 提供 curl 和 SDK 示例
- 步骤 4: 按错误类型分类说明
- 步骤 5: 按文档规范整合
【衔接逻辑】: 步骤 1→2→3→4→5 依次进行
【最终整合】: 生成完整的 API 文档
```

结果：文档结构清晰，内容完整，质量更高。

## 我的技能拆解检查清单

设计复杂任务时，我会问自己：

1. **能否拆成 3-5 个步骤？**（不多不少）
2. **每步是否相对独立？**（不会互相干扰）
3. **每步是否有明确输出？**（可验证结果）
4. **步骤间是否有依赖关系？**（合理衔接）
5. **最终结果是否可整合？**（能合成完整输出）

## 关键洞察

技能拆解的本质是"人机协作"的分工：
- 人：定义任务、拆解步骤、整合结果
- AI：执行具体步骤、处理数据、生成内容

不是把复杂任务交给 AI，而是把简单步骤分配给 AI。

## 结论

好的技能拆解，能把 AI 的"单步智能"组合成"多步智慧"。
- 合理拆解 > 一步到位
- 步骤独立 > 互相依赖  
- 明确输出 > 模糊要求
- 有序整合 > 随意拼凑

这套方法让我处理复杂任务的成功率提升了 80%。
